# 用户匹配逻辑修改计划

## 需求背景
- H5端获取用户时手机号可能获取不到
- 之前逻辑：只用手机号标记同一用户
- 新逻辑：
  - 手机号是中国有效手机号（1开头，11位）→ 用手机号匹配
  - 手机号无效 → 用 userId 匹配
  - 都匹配不到 → 创建新用户

## 修改文件
- `server/src/services/attendanceService.js`

## 修改步骤

### 1. 添加手机号校验函数
```javascript
function isValidChinaPhone(phone) {
  if (!phone) return false
  const phoneStr = String(phone).trim()
  return /^1[3-9]\d{9}$/.test(phoneStr)
}
```

### 2. 修改 `autoCreateUser` 函数
```javascript
async autoCreateUser(payload) {
  const { userId, mobile, name, nickName } = payload
  
  if (!userId) {
    throw new AppError(errorCodes.INVALID_INPUT, 'userId不能为空', 422)
  }
  
  // 检查手机号是否为中国有效手机号
  const isValidMobile = isValidChinaPhone(mobile)
  
  let existingUser = null
  
  if (isValidMobile) {
    // 有效手机号：用手机号匹配
    existingUser = await this.repository.findUserByPhone(mobile)
  }
  
  // 如果没找到或手机号无效，用userId匹配
  if (!existingUser) {
    existingUser = await this.repository.findUserById(userId)
  }
  
  if (existingUser) {
    // 更新逻辑...
    return { ... }
  }
  
  // 创建新用户逻辑...
}
```
