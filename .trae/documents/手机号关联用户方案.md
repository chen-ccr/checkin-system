# 用户手机号关联方案

## 背景
- H5打卡系统使用App的SDK获取用户（JSSDK），与后台管理系统是两个独立用户体系
- 需要通过**手机号**作为关联标记，同一手机号视为同一用户

## 修改范围

### 1. 数据库层 (server/src/db/migrate.js)
- `users`表新增 `phone` 字段（VARCHAR(32), UNIQUE）

### 2. 后端API层 (server/src/routes/v1.js)
- POST `/admin/users` 新增手机号保存
- GET `/admin/users` 返回数据包含 phone
- GET `/admin/bootstrap` 返回数据包含 phone 字段
- POST `/h5/user/auto-create` H5端自动创建用户接口（按手机号查找，不存在则创建）

### 3. 后台管理前端 (admin/src/App.vue)
- 用户表单新增「手机号」输入框
- 用户列表表格新增「手机号」列
- 用户导入功能：支持Excel导入，字段：部门、姓名、手机号（必填）

### 4. H5前端 (h5/src/utils/auth.js)
- 确保 `mobile` 字段正确传递

### 5. H5打卡逻辑 (h5/src/App.vue)
- 打卡前调用 `POST /h5/user/auto-create` 接口
- 传入：userId（App采集的ID）、mobile（手机号）、name（姓名）
- 如果手机号已存在，返回现有用户信息；如果不存在，创建新用户（部门=未知）

## 接口设计

### POST /h5/user/auto-create
**请求体：**
```json
{
  "userId": "app_user_id_123",
  "mobile": "13800138000",
  "name": "张三"
}
```

**响应：**
```json
{
  "code": "OK",
  "data": {
    "userId": "user_xxx",
    "name": "张三",
    "mobile": "13800138000",
    "departmentId": 1,
    "isNew": true
  }
}
```

## 数据库迁移
```sql
ALTER TABLE users ADD COLUMN phone VARCHAR(32) NULL UNIQUE AFTER name;
```
