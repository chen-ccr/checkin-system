# 项目线上部署计划

## 项目架构
当前项目使用 docker-compose 部署，包含4个服务：
- **server**: Node.js API后端 (端口3001)
- **h5**: Vue.js H5打卡页面 (80:8091)
- **admin**: Vue.js 后台管理 (80:8090)
- **mysql**: MySQL数据库 (端口3306)

## 线上部署需要的条件

### 1. 服务器要求
- **Linux服务器**（阿里云ECS或其他）
- **已安装Docker和Docker Compose**
- **公网IP或域名**

### 2. 数据库方案（二选一）

#### 方案A：使用云数据库RDS MySQL（推荐）
- 购买阿里云RDS MySQL实例
- 将当前数据库数据导出并导入到RDS
- 修改docker-compose环境变量指向RDS

#### 方案B：保留服务器上的MySQL
- 如果服务器配置足够，可以直接运行MySQL容器
- 建议：至少2核4G内存

### 3. 域名（可选）
- 备案后的域名
- 可用于HTTPS访问

## 部署步骤

### 第一步：准备服务器
- 购买或使用现有Linux服务器
- 安装Docker: `curl -fsSL https://get.docker.com | sh`
- 安装Docker Compose: `apt install docker-compose`

### 第二步：配置数据库
- 购买阿里云RDS MySQL（或使用服务器本地MySQL）
- 导出本地数据: `mysqldump -u root -p checkin > checkin.sql`
- 导入到RDS

### 第三步：部署项目
```bash
# 1. 克隆代码到服务器
git clone <项目地址> /opt/checkin-system

# 2. 修改环境变量
cd /opt/checkin-system
# 修改 docker-compose.yml 中的数据库地址为RDS地址

# 3. 启动服务（不包含mysql容器，因为用RDS）
docker-compose up -d --build server h5 admin
```

### 第四步：配置Nginx反向代理（可选）
- 将API、H5、管理后台配置到不同路径或子域名
