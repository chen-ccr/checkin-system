# H5页面初始化问题修复计划

## 问题分析

### 问题1：后台管理昵称为空
- `auto-create`接口没有接收`nickName`参数
- H5传递了`nickName`但后端没用

### 问题2：H5页面进入时部门显示未知
- H5页面初始化时只调用`ensureLogin()`获取用户信息
- 没有调用后端接口查询用户在系统中的实际信息（部门等）
- 只有点击打卡时才调用`auto-create`，获取到部门信息后更新页面

## 解决方案

### 1. 修改 `/h5/user/auto-create` 接口 (server/src/routes/v1.js)
- 接收并传递 `nickName` 参数

### 2. H5页面初始化时调用用户信息接口
- 新增接口 `GET /h5/user/info` 或修改现有逻辑
- 页面加载时用手机号查询用户在系统中的信息
- 如果用户存在，返回昵称和部门信息
- 如果用户不存在，返回空，由打卡时创建

### 3. H5初始化流程调整
- `onMounted` 获取用户信息后
- 调用 `/h5/user/info?mobile=xxx` 获取系统用户信息
- 更新 `user.value.departmentName` 和 `user.value.nickname`

## 接口设计

### GET /h5/user/info?mobile=xxx
返回：
```json
{
  "code": "OK",
  "data": {
    "userId": "681032c816d5120019c7e713",
    "name": "谢*",
    "nickname": "大柚子",
    "mobile": "1",
    "departmentId": 1,
    "departmentName": "融媒体中心",
    "exists": true
  }
}
```

如果用户不存在，返回 `exists: false`
