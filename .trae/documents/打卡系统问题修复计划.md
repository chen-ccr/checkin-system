# 打卡系统问题修复计划

## 项目概述
- **项目名称**: 融媒体中心考勤打卡系统
- **技术栈**: Vue 3 + Node.js + MySQL + 高德地图
- **架构**: 前后端分离，包含 H5 端和后台管理系统

## 问题清单与修复方案

### 问题1: 围栏管理地图定位问题
**现象**: 地图打开时位置偏差大，未定位到当前位置
**根因分析**:
- [admin/src/App.vue:293-296](admin/src/App.vue#L293-L296) 地图初始化时使用硬编码坐标 `center: [115.89925, 28.68503]`
- 未调用高德地图定位 API 获取用户当前位置

**修复方案**:
1. 添加高德地图定位插件 `AMap.Geolocation`
2. 地图初始化后自动获取当前位置并定位
3. 添加文字搜索功能（使用 `AMap.PlaceSearch` 插件）

**涉及文件**:
- `admin/index.html` - 添加定位和搜索插件
- `admin/src/App.vue` - 修改 `initFenceMap` 函数

---

### 问题2: 后台管理系统背景白边问题
**现象**: 背景两边有白边，未填充满
**根因分析**:
- [admin/src/App.vue:745-753](admin/src/App.vue#L745-L753) `.login-page` 使用 `min-height: 100vh` 但可能有默认 body 边距
- 需要检查全局样式是否有 `margin: 0` 和 `padding: 0`

**修复方案**:
1. 添加全局样式重置，确保 body 无边距
2. 确保 `.login-page` 背景覆盖整个视口

**涉及文件**:
- `admin/src/App.vue` - 修改全局样式

---

### 问题3: PC端内容区域过小问题
**现象**: PC端打开内容挤在很小区域，未做响应式适配
**根因分析**:
- 登录页 `.login-container` 有 `max-width: 420px`，这是合理的
- 登录后的内容区域 `.card` 等没有设置最大宽度限制
- 需要区分 PC 和手机端显示

**修复方案**:
1. 为登录后的主内容区域添加 `max-width` 和 `margin: 0 auto`
2. 使用 CSS 媒体查询实现响应式布局
3. PC端显示更宽敞的布局，手机端保持紧凑

**涉及文件**:
- `admin/src/App.vue` - 添加响应式样式

---

### 问题4: 定位偏移计算问题（4000多米偏差）
**现象**: 用户经纬度和围栏几乎同一位置，但提示偏移4000多米
**根因分析**:
1. **坐标系不一致问题**:
   - 高德地图使用 **GCJ-02** 坐标系（国测局坐标）
   - 手机原生 GPS 返回 **WGS-84** 坐标系
   - 两种坐标系在中国境内偏差约 100-2000 米不等
   
2. **经纬度顺序检查**:
   - 高德地图 API 使用 `[lng, lat]` 顺序
   - 数据库存储的是 `lat`（纬度）和 `lng`（经度）
   - 需确认存储和计算时顺序正确

3. **代码检查**:
   - [server/src/utils/geo.js:1-14](server/src/utils/geo.js#L1-L14) Haversine 公式正确
   - [admin/src/App.vue:299-305](admin/src/App.vue#L299-L305) 地图点击获取坐标正确
   - [h5/src/utils/location.js:46-54](h5/src/utils/location.js#L46-L54) 从 SDK 获取坐标

**修复方案**:
1. 在后端添加坐标系转换函数（WGS-84 转 GCJ-02）
2. 或者在前端获取位置时进行坐标转换
3. 添加调试日志确认坐标值

**涉及文件**:
- `server/src/utils/geo.js` - 添加坐标转换函数
- `server/src/services/attendanceService.js` - 在距离计算前进行坐标转换

---

### 问题5: H5端打卡时间显示格式问题
**现象**: 时间显示为 `2026-03-26T08:59:00.000Z` 格式
**根因分析**:
- [server/src/services/attendanceService.js:798](server/src/services/attendanceService.js#L798) `punchedAt` 直接返回数据库 DATETIME 值
- 数据库返回的时间可能是 ISO 格式字符串
- H5端未对时间进行格式化

**修复方案**:
1. 在后端返回数据时格式化时间为 `YYYY-MM-DD HH:mm:ss`
2. 或在 H5 端添加时间格式化函数

**涉及文件**:
- `server/src/services/attendanceService.js` - 格式化 `punchedAt`
- `h5/src/App.vue` - 添加时间格式化显示

---

### 问题6: 打卡时段逻辑问题
**现象**: 打卡逻辑可能不符合预期
**当前逻辑分析**:
- [server/src/services/ruleEngine.js:11-23](server/src/services/ruleEngine.js#L11-L23) `resolveCurrentPunchIndex` 要求当前时间必须在时段内
- 例如：设置 9:00-12:00，则只有在 9:00-12:00 之间才能打卡

**用户期望逻辑**:
- 时段外打卡更合理
- 例如：9:00-12:00 的班次
  - 9:00 之前打卡 → 算上班卡（正常）
  - 9:00-12:00 打卡 → 可能算迟到
  - 12:00 之后打卡 → 算下班卡或下一个节点

**修复方案**:
1. 修改打卡时段判断逻辑
2. 上班卡：在时段开始时间之前或时段内打卡
3. 下班卡：在时段结束时间之后或时段内打卡
4. 根据打卡时间自动判断是哪个节点

**涉及文件**:
- `server/src/services/ruleEngine.js` - 修改 `resolveCurrentPunchIndex` 逻辑
