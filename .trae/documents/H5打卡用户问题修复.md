# H5打卡用户问题修复计划

## 问题分析
1. H5页面打卡时提示"用户信息加载中，请稍后重试"
2. 原因：`handleCheckin`在`user.value?.userId`为空时直接return，不尝试获取用户
3. JSSDK获取用户的数据格式检查：返回`res.code`, `res.data.accountId`, `res.data.mobile`, `res.data.user.realName`

## 修复方案

### 1. 修改 handleCheckin 函数 (h5/src/App.vue)
- 如果`user.value`为空，调用`ensureLogin()`获取用户
- 只要有手机号就可以进行打卡
- 调用`/h5/user/auto-create`接口创建用户

### 2. 修改后端 auto-create 接口
- 确保能通过手机号查找用户
- 如果找不到，自动创建用户（部门=未知）

## 具体修改

### h5/src/App.vue - handleCheckin
```javascript
async function handleCheckin() {
  // 如果没有用户信息，尝试获取
  if (!user.value) {
    try {
      user.value = await ensureLogin()
    } catch (e) {
      msg.value = '获取用户信息失败'
      return
    }
  }

  if (loading.value) return
  loading.value = true
  msg.value = '正在处理...'

  try {
    // 确保用户已注册到系统
    let checkinUserId = user.value.userId
    if (user.value.mobile) {
      const autoRes = await api.post('/h5/user/auto-create', {
        userId: user.value.userId,
        mobile: user.value.mobile,
        name: user.value.name
      })
      if (autoRes.data.code === 'OK') {
        checkinUserId = autoRes.data.data.userId
      }
    }

    // 继续定位和打卡流程...
  }
}
```
