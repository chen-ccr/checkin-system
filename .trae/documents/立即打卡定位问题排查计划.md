# 立即打卡定位问题排查计划

## 问题描述
在App内打开H5页面时，点击"立即打卡"后一直显示"正在定位并提交..."，说明 `getLocation()` 函数一直没有返回。

## 相关代码分析

### 1. getLocation 函数 (`h5/src/utils/location.js`)
```javascript
export function getLocation() {
  return new Promise((resolve, reject) => {
    if (window.CP2?.getLocation) {
      // App环境：使用CP2的定位
      window.CP2.getLocation().then(resolve).catch(reject)
    } else {
      // 浏览器环境：使用浏览器定位
      navigator.geolocation.getCurrentPosition(
        pos => resolve({ lat: pos.coords.latitude, lng: pos.coords.longitude }),
        reject
      )
    }
  })
}
```

### 2. handleCheckin 函数 (`h5/src/App.vue`)
- 第290行：设置 `msg.value = '正在定位并提交...'`
- 第292行：调用 `const loc = await getLocation()`
- 如果成功：调用 `checkFence(loc)` 和 `api.post('/checkins', payload)`
- 如果失败（catch）：再次调用 `getLocation()` 并保存离线数据

## 可能的问题原因

### 原因1：window.CP2.getLocation() 可能未定义或无响应
- App中可能没有正确注入 `window.CP2.getLocation`
- 或者注入的getLocation一直不返回（权限请求、GPS问题等）

### 原因2：定位超时
- GPS定位可能需要较长时间
- 可能需要添加超时处理

### 原因3：catch块中的重复调用
- 如果第一次getLocation()失败，会进入catch块
- catch块又调用getLocation()，如果还失败就会一直卡住

## 修复方案

### 步骤1：添加调试日志
在 `location.js` 中添加console.log，查看window.CP2是否存在，getLocation是否被调用

### 步骤2：添加超时处理
给getLocation添加超时逻辑，比如3秒或5秒超时

### 步骤3：简化catch块逻辑
catch块中不需要再次获取定位，直接使用离线模式保存即可

### 步骤4：验证修复
在App中测试定位功能

## 实施步骤
1. 修改 `h5/src/utils/location.js` 添加调试和超时
2. 修改 `h5/src/App.vue` 的handleCheckin catch块
3. 重新构建并测试
