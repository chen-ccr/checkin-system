# 打卡系统问题修复计划（第二轮）

## 问题清单与修复方案

### 问题1: 后台管理系统内容宽度太小
**现象**: PC电脑上看，白色背景的内容区域只占了一小块
**根因分析**:
- 登录后的主内容区域没有设置合适的最小宽度
- `.page` 类设置了 `max-width: 1200px`，但没有 `min-width`
- 卡片内容在PC端显得过于紧凑

**修复方案**:
1. 为登录后的主内容区域设置 `min-width` 和更大的 `max-width`
2. 调整卡片在PC端的内边距
3. 优化表格和表单在PC端的显示宽度

**涉及文件**:
- `admin/src/App.vue` - 修改 `.page` 和相关样式

---

### 问题2: 围栏管理地图加载慢、搜索无效、定位失败
**现象**: 
- 地图加载出来很慢
- 搜索功能没用
- 没有定位到当前位置

**根因分析**:
1. **地图加载慢**: 高德地图 API 从国内服务器加载，可能网络问题
2. **搜索无效**: `placeSearch` 初始化后没有正确绑定城市范围
3. **定位失败**: 浏览器定位需要 HTTPS 或 localhost，且需要用户授权

**修复方案**:
1. 添加地图加载状态提示
2. 修复搜索功能，添加城市参数
3. 添加定位失败时的回退方案（使用 IP 定位）
4. 添加错误提示信息

**涉及文件**:
- `admin/src/App.vue` - 修改 `initFenceMap` 和 `searchPlace` 函数

---

### 问题3: H5端状态显示为英文
**现象**: 显示 "NORMAL"、"PENDING"、"LATE" 等英文状态
**根因分析**:
- 模板中直接使用 `{{ node.status }}` 和 `{{ item.status }}`
- 没有进行中文翻译

**修复方案**:
1. 添加状态翻译函数 `translateStatus(status)`
2. 在模板中使用翻译函数

**涉及文件**:
- `h5/src/App.vue` - 添加翻译函数并修改模板

---

### 问题4: 考勤汇总柱状图间距问题
**现象**: 人数很少时柱状图间距非常大，需要拖动查看
**根因分析**:
- `.chart-item` 固定宽度 72px
- `.chart-area` 设置 `min-width: 500px`
- 当数据少时，`justify-content: space-around` 导致间距过大

**修复方案**:
1. 改为固定每组柱子的间距（如 20px gap）
2. 每个柱子组宽度自适应
3. 整体可滚动，但内部间距固定

**涉及文件**:
- `h5/src/App.vue` - 修改柱状图样式
