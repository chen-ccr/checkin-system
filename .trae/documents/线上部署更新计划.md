# 线上部署更新计划

## 现有环境分析

### docker-compose.prod.yml 配置
- **server**: 后端服务，镜像 `checkin-server:latest`，端口 3001
- **h5**: H5前端，镜像 `checkin-h5:latest`，端口 8091
- **admin**: 管理后台，镜像 `checkin-admin:latest`，端口 8090
- 数据库使用远程服务器：110.42.214.127:13306

### docker-compose.yml 配置
- 与 prod 版本类似，但使用 `${IMAGE_TAG:-latest}` 变量
- 支持通过环境变量配置镜像标签

## 部署更新步骤

### 步骤1: 检查服务器上现有的镜像和容器
```bash
# 查看所有镜像
docker images

# 查看正在运行的容器
docker ps -a
```

### 步骤2: 清理多余的旧镜像（可选）
```bash
# 删除未使用的镜像
docker image prune -a

# 或者删除指定镜像
docker rmi <image_id>
```

### 步骤3: 本地构建新镜像
```bash
# 使用 prod 配置构建（不推送，直接构建）
docker-compose -f docker-compose.prod.yml build

# 或者带标签构建
IMAGE_TAG=prod docker-compose build
```

### 步骤4: 导出镜像并上传到服务器
```bash
# 导出镜像为 tar 文件
docker save -o checkin-server.tar checkin-server:latest
docker save -o checkin-h5.tar checkin-h5:latest
docker save -o checkin-admin.tar checkin-admin:latest

# 上传到服务器
scp *.tar user@server:/path/to/project/

# 在服务器上导入镜像
docker load -i checkin-server.tar
docker load -i checkin-h5.tar
docker load -i checkin-admin.tar
```

### 步骤5: 在服务器上重启服务
```bash
cd /path/to/project

# 停止旧服务
docker-compose -f docker-compose.prod.yml down

# 启动新服务
docker-compose -f docker-compose.prod.yml up -d
```

---

## 简化方案：使用 Docker Hub 或私有仓库

如果配置了镜像仓库，可以：
1. `docker-compose -f docker-compose.prod.yml push` 推送镜像
2. 在服务器上 `docker-compose -f docker-compose.prod.yml pull` 拉取新镜像
3. `docker-compose -f docker-compose.prod.yml up -d` 重启服务
