# 围栏管理系统问题修复计划

## 问题分析

### 问题1: 围栏编辑后列表不更新
**现象**: 第一次编辑围栏后列表会更新，后续编辑不更新，需要刷新页面
**根因分析**:
- `loadBootstrap()` 函数中直接赋值 `bootstrap.value = {...}` 可能不会触发 Vue 响应式更新
- 需要使用 `Object.assign` 来确保响应式更新

### 问题2: 地图不定位到当前位置
**根因分析**:
- `initFenceMap()` 只在页面加载时通过 `setTimeout` 调用一次
- 如果围栏管理标签页不是默认显示的，地图容器可能尺寸为0，导致初始化失败
- 定位插件可能加载失败但没有错误处理

### 问题3: 搜索后地图不定位
**根因分析**:
- `doSearch()` 中的 `placeSearch.search` 回调执行后，地图可能还没初始化完成
- 或者 `fenceMap.setCenter` 调用时地图状态不对

---

## 修复方案

### 修复1: 围栏列表更新
```javascript
// 原来
bootstrap.value = { ... }

// 修改为
Object.assign(bootstrap.value, {
  departments: data.departments || [],
  roles: data.roles || [],
  geofences: data.geofences || [],
  shiftRules: data.shiftRules || []
})
```

### 修复2: 地图定位
1. 添加 `watch` 监听 `activeTab`，当切换到围栏标签时初始化地图
2. 确保地图容器有尺寸后再初始化
3. 改进定位错误处理

### 修复3: 搜索定位
1. 在 `doSearch` 中添加延时确保地图准备好
2. 添加搜索状态的 loading 提示
